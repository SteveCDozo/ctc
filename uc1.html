Vulnerability & Penetration Testing Platform For Cloud Applications

TARGETED ISSUE

Just how secure is your system?

Companies are eager to implement new technologies into their systems on the cloud. IT teams release code quickly and frequently. They may possibly overlook some subtle bugs or loopholes in the newly updated codebase.
Risks Due to These Bugs
  • Information Loss
  • Monetary Loss
  • Damage to Reputation
  • Customer Dissatisfaction
  • Loss of Customers
SOLUTION PURPOSE

We've got you covered.

It's essential to test cloud applications prior to deployment to ensure security and optimal performance. This solution allows a company to have its code analyzed by experienced developers around the world. A company can feel more reassured that its system is safe and secure.

The platform will transform traditional security testing to a data-driven vulnerability management tool. It will provide a complete and actionable vulnerability report including an executive summary, a listing of risk ratings, and remediation recommendations to the client.
SOLUTION PROCESS

Simple and easy

  • 1
    Client Submits a Program

    A client submits the program Virtual Machine image they want to be analyzed for vulnerabilities

  • 2
    Platform Runs a Malware Check

    The platform will run a quick check and then upload the image to the sandbox account and publish the task

  • 3
    Experts Begin Testing

    Cybersecurity experts will login, select the task, and start testing the client's program

  • 4
    Experts Compete to Be the First to Find an Issue

    The first expert to successfully identify an issue will be rewarded and a record of this transaction will be added to the blockchain

  • 5
    Client is Notified

    The client is sent a notification containing the report for the issue

For any average user

Basic levels of testing, even for small portions of code

Ben's new cloud application is scheduled to release in two weeks. With such limited time left, he decides to try our system to quickly find any security problems. He signs up for a regular account, creates an Amazon Machine Image, and copies it to the provided sandbox account. He pays 6 V Tokens to have it tested for three days and waits for his testing request to be approved. After 3 days, he receives an email notifying him the testing has started. Now Ben only needs to wait for the results. After three days of testing, Ben receives a complete report with an executive summary, test data visualizations, a listing of risk ratings, and recommended firewall and insurance plans from partners.

For those with more testing needs

Faster request reviews, daily reports, and the ability to search for testers

Alex, a software developer, has just developed a new program on Microsoft Azure. He is a premium user on our platform, so he can start testing his program quickly and receive testing results daily. He creates a VM image from Azure and uploads it to our platform. He decides to have it tested by five-star ranked testers for two weeks, so he pays 14 V Tokens. The platform runs the premium vulnerability scanner, generates a report, and sends it to Alex. These issues will not be reported to him again and testers won't be rewarded for these issues in the future. The testing gets approved after one day and reports get sent to him everyday. After a week, Alex decides to stop the testing, so 7 V Tokens are returned to him. and a final report is sent to him.

For your development team

Adjust access rights and extend testing to your own programmers too

Jane works in a global company as the chief of technology. She decides to use the platform to test newly developed software. Instead of only having the testing team analyze the software, she creates accounts for all of her company's programmers around the world and set up a group for read-only access to the code. A notice is sent through the company email asking those programmers to test this new software. A report of full test results is sent to Jane and the development team.

SECURITY

How is security guaranteed?

Professional Crowd

All of our crowd members must pass proficiency tests before participating in any testing assignments. They are also required to sign confidentiality agreements before accepting tasks.

Initial Security Scan

An automated test will be run against every machine image uploaded to the sandbox accounts. If malware is found, the client will be sent a warning and the instance will be deleted. Since sandbox accounts cannot access the platform database or source code, the platform will not be harmed in the event of a malware attack.

Amazon Organizations & IAM

AWS Organizations and IAM services allow for access control with different user groups and policies. Access can be granted for a specific date range, different AWS service APIs, and specific resources.

Hyperledger Technology

Hyperledger is an open source blockchain technology which supports private transactions and confidential contracts. The blockchain will record every action taken on clients’ uploaded programs, providing full transparency. The blockchain technology is also used to ensure the safety of the transaction used through smart contracts for the reward process.

OPERATION & FEES

What are V Tokens?

The VPTP (Vulnerability & Penetration Testing Platform) Token, or V Token, is a cryptocurrency designed to incentivize community members to perform actions that create value within the community. The platform encourages users to test programs and the rewarding system guarantees that users who find problems are rewarded by V Tokens.
Uses of V Tokens
  • Submit the cloud application to be tested
  • Upgrade an account to premium or enterprise
  • Record the activity on your application
  • Pay for experts to find issues
